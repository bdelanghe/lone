# Accessibility Statement Template

## Commitment

We are committed to providing an accessible experience for all users. We aim to
meet WCAG 2.1 AA standards across our product and documentation.

## What We've Done

- Implemented semantic HTML and ARIA where appropriate
- Conducted automated accessibility checks
- Performed manual testing for keyboard and screen reader use

## Known Issues

- List any known accessibility gaps
- Provide planned remediation timelines
- Offer temporary workarounds when possible

## Third-Party Content

Some embedded content or integrations may be controlled by third parties. We
work with partners to improve accessibility where possible, but cannot guarantee
full compliance for external content.

## Assistive Technology Compatibility

- Browsers tested: Chrome, Firefox, Safari, Edge
- Screen readers tested: VoiceOver, NVDA, JAWS
- Keyboard navigation: fully supported
